Claude 연동 포탈

Claude가 GPT 이미지로도 그리게 하기

OpenAI Codex CLI를 ChatGPT 구독 로그인만으로 연동하면, Claude가 필요할 때 GPT 이미지(gpt-image) 생성을 도구처럼 부려 쓸 수 있습니다. API 키 발급도, 추가 결제도 필요 없습니다.

약 5분 소요 Windows · macOS 공통 API 키 불필요
시작하기 먼저 원리 보기

한눈에 보는 작동 원리

Codex CLI는 OpenAI의 터미널 에이전트입니다. ChatGPT 계정으로 로그인(auth)하면 구독에 포함된 권한으로 gpt-image 이미지 생성을 쓸 수 있고, Claude는 이 CLI를 터미널 도구로 호출합니다.

Claude

당신이 자연어로 지시

Codex CLI

ChatGPT 구독으로 인증된 터미널 도구

gpt-image PNG

지정한 폴더에 저장

핵심은 auth 방식입니다. API 키를 발급받아 종량 과금하는 게 아니라, codex login으로 이미 쓰고 있는 ChatGPT 구독 계정에 로그인해 그 사용량 안에서 이미지를 생성합니다.

시작 전 준비물

세 가지면 충분합니다. Windows·macOS 모두 동일합니다.

Node.js 18+

터미널에서 node --version으로 확인. 없으면 nodejs.org에서 설치.

ChatGPT 유료 구독

Plus · Pro · Team 계정. 이 구독 인증으로 gpt-image를 씁니다. API 키는 필요 없습니다.

Claude Code

연동해서 부려 쓸 Claude. 이미 쓰고 있다면 그대로 OK.

5단계로 연동 완성

설치 → 로그인 → 테스트 → Claude 연동 순서입니다.

1

Codex CLI 설치

터미널(PowerShell / 터미널.app)에서 한 줄이면 됩니다. Windows·macOS 공통.

npm install -g @openai/codex

설치 확인:

codex --version
2

ChatGPT 계정으로 로그인 (auth)

여기가 이 가이드의 핵심입니다. API 키 대신 구독 계정 로그인으로 인증합니다.

codex login

→ 브라우저 창이 열리면 평소 쓰는 ChatGPT 계정으로 로그인합니다. "Sign in with ChatGPT"를 선택하세요.

로그인은 한 번만 하면 됩니다. 인증 정보가 로컬(~/.codex/)에 저장돼 이후 자동으로 쓰입니다. API 키를 만들거나 결제 수단을 등록할 필요가 없습니다.
3

첫 이미지 생성 테스트

프롬프트는 명령어에 직접 쓰지 않고, 텍스트 파일로 저장해서 넘기는 방식이 안전합니다. 먼저 메모장 등으로 prompt.txt 파일을 만들어 이렇게 적습니다.

귀여운 아기 고양이가 노트북으로 코딩하는 일러스트를 생성해줘.
Save as cat.png

그 다음 실행합니다.

Windows (PowerShell) 
Get-Content prompt.txt | codex exec --full-auto --skip-git-repo-check -C "C:\작업폴더" -
macOS 
cat prompt.txt | codex exec --full-auto --skip-git-repo-check -C ~/작업폴더 -

약 30초 뒤 -C로 지정한 폴더에 cat.png가 생기면 성공입니다.

한글 프롬프트는 반드시 이렇게 "파일로 저장해서" 넘기세요. 명령어 뒤에 한글 문장을 바로 붙여 쓰면 글자가 깨져 실패합니다. 위 명령은 "파일 내용을 읽어서 codex에 흘려 넣어라"는 뜻이고, 맨 끝의 - 기호까지 포함해 통째로 복사하면 됩니다. 그리고 프롬프트 안에 "Save as 파일명.png"를 꼭 넣어야 내 작업 폴더에 저장됩니다(없으면 깊숙한 임시 폴더에 들어가 찾기 번거로움).
4

Claude와 연동 — 자연어로 시키기

별도 설정이 필요 없습니다. Codex CLI가 로그인돼 있으면, Claude Code에게 이렇게 말하는 것으로 연동이 끝납니다.

codex로 GPT 이미지 생성해줘 — 미니멀한 흰 배경의 커피 브랜드 포스터,
한글 타이틀 "아침의 온도", coffee_poster.png로 저장

→ Claude가 프롬프트 파일 작성 → codex exec 실행 → 완료 감지·프로세스 정리 → 결과 확인까지 알아서 처리합니다. 여러 장이면 순차로 돌리고, 아래 "매달림 현상"도 Claude가 스스로 처리합니다.

이 방식의 좋은 점 — ComfyUI(로컬 GPU)와 GPT 이미지(클라우드)를 Claude가 한 자리에서 골라 쓰는 이미지 작업 환경이 됩니다. 사진 같은 실사·한글 타이포는 GPT, 캐릭터·스타일 일관성은 ComfyUI 식으로.
5

결과 확인

지정한 폴더에 PNG가 생겼는지 확인합니다. 파일이 안 보이면:

기본 저장 위치(Save as 미지정 시):
~/.codex/generated_images/{세션-id}/
여기까지 되면 연동 완료. 이제 Claude에게 "codex로 이미지 만들어줘"라고만 하면 됩니다.

실전 팁 — 실제로 겪은 것들

수십 장을 뽑아보며 확인한 주의사항입니다.

매달림(hang) 현상 — --full-auto는 이미지를 저장한 뒤에도 프로세스가 스스로 끝나지 않고 매달릴 수 있습니다(1시간 이상 확인됨). 배치로 여러 장을 돌릴 땐 "출력 PNG가 생기면 프로세스를 강제 종료하고 다음으로"(poll-then-kill) 패턴이 필수입니다. Claude에게 시키면 이 처리를 알아서 합니다.
병렬 실행 금지 — 동시에 여러 개 돌리면 같은 세션 파일을 덮어써 꼬입니다. 여러 장은 순차 실행이 정답입니다. 한 장에 약 30초 걸립니다.
레퍼런스 이미지로 톤 맞추기 — -C 작업 폴더에 참고할 PNG를 넣고 프롬프트에 그 파일명을 언급하면, 톤·구도를 따라간 일관된 시리즈를 만들 수 있습니다. (WIZMO 차량 시리즈 4장으로 검증)
사용량 — 이미지 한 장에 구독 사용량(토큰)을 꽤 씁니다(약 6만 토큰). 무한정은 아니니 대량 생성은 ComfyUI 쪽이 경제적입니다.

GPT 이미지, 무엇에 강한가

로컬 ComfyUI와 역할을 나누면 좋습니다.

한글 텍스트

포스터·배너 속 한글 타이포가 깨지지 않고 또렷하게 들어갑니다. 로컬 모델 대비 최대 강점.

UI 목업

앱 화면·대시보드·웹 페이지 목업을 그럴듯하게 뽑습니다.

포토리얼

실사풍 인물·제품·공간 사진 품질이 좋습니다.

인포그래픽

데이터 기반 도표·다이어그램형 이미지에 강합니다.

한계 — 추상적 컨셉아트는 전형적인 "AI 그림" 느낌으로 흐르기 쉽고, 그림체가 로컬 모델(ComfyUI)과 달라 시리즈를 섞으면 톤이 튑니다. 캐릭터 일관성이 필요한 작업은 로컬 파이프라인이 낫습니다.

문제 해결

막히면 여기부터.

한글 프롬프트가 이상하게 나감 / 실패 — 명령어 뒤에 한글 문장을 직접 붙여 쓰면 글자가 깨집니다. Step 3처럼 프롬프트를 prompt.txt 파일로 저장하고, 안내된 명령을 통째로 복사해서 실행하세요.
이미지가 어디 저장됐는지 모르겠음 — 프롬프트에 "Save as 파일명.png"를 안 넣은 경우입니다. ~/.codex/generated_images/ 밑 최신 세션 폴더를 확인하고, 다음부터는 Save as를 넣으세요.
로그인 창이 안 뜸 / 인증 만료 — codex login을 다시 실행하세요. 회사 프록시·VPN이 브라우저 인증을 막는 경우 잠시 끄고 로그인합니다.
보안 — codex login 인증 정보는 내 컴퓨터 ~/.codex/에 저장됩니다. 공용 PC에서는 쓰고 나서 codex logout으로 로그아웃하세요. 계정 비밀번호를 Claude나 스크립트에 적을 일은 없습니다.
