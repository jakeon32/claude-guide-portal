Claude 연동 포탈

Claude가 Figma를 직접 다루게 하기

figma-console MCP를 연결하면 Claude가 Figma에서 디자인을 읽고, 프레임·컴포넌트를 만들고, 변수·토큰을 관리하고, 스크린샷으로 확인까지 합니다. Windows·macOS 공통으로 따라 하세요.

약 10분 소요 Windows · macOS 공통 5단계
시작하기 먼저 원리 보기

한눈에 보는 작동 원리

Claude와 Figma는 MCP(Model Context Protocol — Claude에 외부 프로그램을 연결해 주는 표준 연결 방식) 서버와 Figma 플러그인이 다리 역할을 해서 연결됩니다.

Claude

당신이 자연어로 지시

figma-console MCP

npx(Node.js에 함께 깔리는 실행 도구)로 뜨는 서버(브리지)

Figma Desktop

Bridge 플러그인이 반영

Figma 데스크톱에 설치한 Desktop Bridge 플러그인이 내 컴퓨터 안(localhost 포트 9223~9232)에서 MCP 서버와 실시간으로 통신합니다. 그래서 인터넷 서버를 거치지 않고, 내 Figma에 바로 반영됩니다.

시작 전 준비물

네 가지만 있으면 됩니다. Windows·macOS 모두 동일합니다.

Node.js 18+

터미널에서 node --version으로 확인. 없으면 nodejs.org에서 설치.

Figma Desktop 앱

이 가이드의 플러그인(manifest 가져오기)은 웹 브라우저 버전이 아닌 데스크톱 앱에서만 설치됩니다.

MCP 클라이언트

Claude Code(CLI) 또는 Claude Desktop. Cursor·Windsurf도 가능.

Figma 계정

개인 액세스 토큰을 발급할 수 있는 로그인 계정.

5단계로 연동 완성

Windows와 macOS 명령·경로를 각 단계에 함께 적었습니다.

1

Figma 개인 액세스 토큰 발급

Figma에서 Claude가 쓸 열쇠(토큰)를 만듭니다.

Figma → 오른쪽 위 프로필 → Settings → Security 탭 → Personal access tokens → Generate new token.

설명은 Figma Console MCP로 적고, 아래 권한(스코프)을 설정합니다.

스코프 — File content: Read · File versions: Read · Variables: Read · Comments: Read and write. 목록에 없는 항목은 건너뛰어도 기본 기능은 동작합니다.
토큰은 figd_로 시작하고 딱 한 번만 표시됩니다. 바로 복사해 안전한 곳에 보관하세요.
2

figma-console MCP 서버 등록

사용하는 클라이언트에 맞춰 하나만 하면 됩니다.

가장 쉬운 방법 — Claude에게 맡기기 (명령 편집 불필요)

Claude Code를 쓰고 있다면, 명령을 손으로 만들 필요 없이 Claude에게 아래처럼 말하면 됩니다. 문장 끝에 Step 1에서 복사해둔 토큰(figd_…)을 그대로 붙여넣으세요.

figma-console MCP를 등록해줘. 범위는 user, ENABLE_MCP_APPS는 true로. 내 Figma 토큰은 이거야:

→ Claude가 알맞은 claude mcp add 명령을 스스로 실행합니다. 복사한 걸 손으로 고칠 일이 전혀 없습니다. ENABLE_MCP_APPS는 부가 기능을 켜는 스위치 — 그대로 두면 됩니다.

또는 직접 명령으로 (한 줄)

figd_YOUR_TOKEN만 실제 토큰으로 바꿔 한 줄 그대로 붙여넣습니다. (Windows·macOS 공통)

claude mcp add figma-console -s user -e FIGMA_ACCESS_TOKEN=figd_YOUR_TOKEN -e ENABLE_MCP_APPS=true -- npx -y figma-console-mcp@latest
인터넷의 다른 안내처럼 명령이 여러 줄로 나뉜 경우, 줄 끝 \(백슬래시)는 macOS·Linux 줄바꿈 기호라 Windows에서는 에러가 납니다. 이 가이드의 블록은 한 줄이라 그대로 됩니다.

Claude Desktop / Cursor — 설정 파일 직접 편집

설정 파일에 아래 블록을 mcpServers 안에 추가합니다.

{
  "mcpServers": {
    "figma-console": {
      "command": "npx",
      "args": ["-y", "figma-console-mcp@latest"],
      "env": {
        "FIGMA_ACCESS_TOKEN": "figd_YOUR_TOKEN",
        "ENABLE_MCP_APPS": "true"
      }
    }
  }
}

설정 파일 위치:

Windows 
Claude Desktop : %APPDATA%\Claude\claude_desktop_config.json
Claude Code(CLI): %USERPROFILE%\.claude.json
Cursor          : %USERPROFILE%\.cursor\mcp.json
macOS 
Claude Desktop : ~/Library/Application Support/Claude/claude_desktop_config.json
Claude Code(CLI): ~/.claude.json
Cursor          : ~/.cursor/mcp.json
~ 는 홈 폴더입니다 — macOS는 /Users/내이름/, Windows는 C:\Users\내이름\. 파일이 없으면 새로 만들면 됩니다. 전체가 올바른 JSON인지(쉼표·괄호) 확인하세요.
3

Figma Desktop Bridge 플러그인 설치

Claude와 Figma를 이어줄 플러그인을 Figma에 넣습니다.

Figma Desktop을 열고 아무 파일이나 연 뒤 (특별한 옵션 없이 평소처럼):

Plugins → Development → Import plugin from manifest… → 아래 manifest.json 선택.

Windows 
C:\Users\내이름\.figma-console-mcp\plugin\manifest.json
macOS 
~/.figma-console-mcp/plugin/manifest.json
= /Users/내이름/.figma-console-mcp/plugin/manifest.json

이 폴더는 MCP 서버가 처음 실행될 때 자동으로 만들어집니다. 안 보이면 Step 2까지 마친 뒤 클라이언트를 한 번 실행해 서버를 띄우세요.

가져온 다음 Plugins → Development → 방금 가져온 플러그인(Figma Desktop Bridge)을 클릭해 실행하면 포트 9223~9232를 스캔해 실행 중인 MCP 서버에 자동 연결됩니다.

Figma는 플러그인 파일을 캐시합니다. 업데이트 후 새 기능이 안 보이면 Import plugin from manifest…를 같은 경로로 다시 실행하세요.
4

MCP 클라이언트 재시작

새 설정을 읽어들이도록 Claude Code(또는 Claude Desktop)를 완전히 종료했다가 다시 실행합니다. Windows·macOS 동일합니다.

5

연결 테스트

Claude에게 아래처럼 말해봅니다.

Figma 상태를 확인해줘

→ 연결 상태가 active(연결됨)로 나오면 성공입니다.

파란 배경의 프레임을 하나 만들어줘

→ Figma에 실제로 프레임이 생기면 쓰기 권한까지 정상입니다.

여기까지 되면 연동 완료. 이제 Claude가 당신의 Figma를 직접 다룹니다.

연동 후 — Figma 작업 스킬 설치

연결만으로도 쓸 수 있지만, figma-component-design 스킬을 깔면 컴포넌트·디자인 시스템 작업이 훨씬 정교해집니다.

이 스킬은 figma-console MCP 위에서 동작하며, 버튼·카드·모달 같은 컴포넌트를 Auto Layout · Variant · Variable · 디자인 토큰까지 제대로 갖춰 만들어 줍니다. (Atoms → Organisms 아토믹 디자인)

A

공식 플러그인에서 설치

Claude Code에서 /plugin 을 실행하고, 공식 마켓플레이스에서 figma 플러그인을 찾아 설치합니다.

/plugin

→ 마켓플레이스 claude-plugins-official → figma 선택 → Install. 이 플러그인이 figma-component-design 스킬을 포함합니다. 설치 후 세션을 다시 시작하면 스킬이 로드됩니다.

B

바로 시켜보기

Figma가 연결된 상태에서 이렇게 말하면 스킬이 자동으로 발동합니다.

버튼 컴포넌트를 Variant(기본/호버/비활성)까지 만들어줘

→ 스킬이 컴포넌트 설계 → 생성 → 스크린샷 검증 순서로 제대로 만들어 줍니다.

스킬 ≠ 연동. figma-console MCP 연결(위 5단계)이 먼저이고, 스킬은 그 위에서 "잘 만들도록" 돕는 전문 지식입니다. 스킬도 시작할 때 먼저 figma_get_status로 연결을 확인합니다.

연동 후 이렇게 씁니다

도구 100개 이상. 자주 쓰는 것들:

디자인 추출

변수·컴포넌트·스타일을 읽어 코드/토큰으로. "이 파일 디자인 토큰 뽑아줘"

디자인 생성·수정

프레임·컴포넌트·레이아웃을 직접 생성/편집. "카드 컴포넌트 만들어줘"

스크린샷 검증

결과를 캡처해 정렬·간격을 확인하고 반복 수정.

토큰·변수 동기화

Figma 변수 ↔ 표준 디자인 토큰 형식(DTCG) JSON·CSS 양방향 싱크.

우리 노하우 — ComfyUI로 생성한 이미지를 Figma에 넣을 땐, 로컬 CORS 프록시(9225)로 이미지를 가져와 figma_set_image_fill로 채웁니다. (천안 셔틀 상품 이미지 제작에 사용)

문제 해결

막히면 여기부터.

"연결 안 됨" / 상태가 비활성 — ① Figma에서 Bridge 플러그인을 다시 실행 ② MCP 클라이언트를 재시작 ③ Figma가 데스크톱 앱인지(웹 아님) 확인 ④ 포트 9223~9232를 막는 방화벽·VPN이 없는지.
읽기는 되는데 생성·수정이 안 됨 — 토큰 스코프가 부족하거나 Read-only 모드입니다. Step 1 스코프를 다시 확인하고, 이 가이드대로(npx 명령으로) 등록했는지 확인하세요.
manifest.json이 안 보임 — Step 2까지 마치고 클라이언트를 한 번 실행해 서버를 띄우면 ~/.figma-console-mcp/plugin/이 자동 생성됩니다. 처음 npx 실행은 다운로드로 몇 분 걸릴 수 있습니다.

보안 수칙

토큰은 열쇠입니다.

figd_ 토큰은 설정 파일에 평문으로 저장됩니다. 그 파일(.claude.json 등)을 남과 공유하거나 깃에 올리지 마세요. 스코프는 필요한 최소로, 의심되면 Figma에서 즉시 폐기 후 재발급하세요.
